# H/W Spec — 원격 릴레이 스위치 (MQTT → 설비 전원 차단)

감지기가 위험을 감지하면 MQTT를 발행하고, **서버(hw-gateway)와 이 릴레이 스위치가 동시에 구독**한다.
릴레이 스위치는 수신 즉시 **전기 접점을 열어 설비 전원을 차단**한다. (사람에겐 별도 알림 — 현재 이메일, 향후 문자)

**정확한 명칭:** 이 장치는 **원격 릴레이 스위치(remote relay switch)** = 네트워크/IP 릴레이, 우리 경우 **MQTT 릴레이 모듈**이다.
- 모터·실린더 같은 기계식 구동기가 **아니고**, 물리적으로 버튼을 누르는 장치(예: 버튼 푸셔)도 **아니다** — 전기 회로를 직접 개폐한다.
- `architecture-overview.md`가 이 역할을 추상적으로 "actuator"라 부르지만, **부품 이름은 원격 릴레이 스위치**다. (actuator = 제어신호로 물리세계에 작용하는 장치의 넓은 범주 — 릴레이도 포함되나 오해 소지 있어 본 문서는 구체명 사용)

## 신호 흐름

```
감지기 ──MQTT(safety/{siteId}/alert)──> Mosquitto(자체 IP)
                                          ├──> 서버(hw-gateway): DB·알림·녹화
                                          └──> [MQTT 릴레이 모듈]  ← 직접 구독
                                                  │ dry contact (COM–NO)
                                                  ▼
                                          전자접촉기 코일(A1/A2)
                                                  │ 주접점
                                                  ▼
                                          설비 3상/단상 전원 차단
```

- **서버 독립(fail-safe) 의도**: 릴레이가 브로커에 직접 붙으므로 서버가 죽어도 차단 동작.
- **핵심 요구**: 커스텀 브로커 IP + 커스텀 토픽(`safety/{siteId}/alert`) **SUBSCRIBE** → 릴레이 접점 개폐.
- **함정**: "MQTT 지원" 광고 제품 다수가 **publish 전용**이거나 **Modbus-TCP 전용**. 아래 표는 subscribe→릴레이 제어가 되는 것만 선별.

## 명칭 정리

| 계층 | 명칭 |
|------|------|
| MQTT 받는 제어 모듈 | **MQTT 릴레이 모듈** / 네트워크(이더넷·WiFi) 릴레이 컨트롤러 / 원격 I/O 릴레이 |
| 실제 전력 차단 부품 | **전자접촉기(마그네틱 컨택터)** 또는 **SSR(솔리드 스테이트 릴레이)** |
| 안전등급 판단부 | **세이프티 릴레이(safety relay)** + 안전접촉기 (하드와이어드) |

## Part B — 전원 스위칭 소자 (전자접촉기 / SSR)

MQTT 릴레이 접점은 무전압·저전류라 모터 전류를 직접 못 흘린다. 접점은 **접촉기 코일만 여자**시키고 대전류는 접촉기 주접점이 담당.

### B-1. 전자접촉기 — LS ELECTRIC METASOL MC
| 모델 | AC-3 | 220V | 380~440V | 대략가(KRW) |
|------|------|------|----------|-------------|
| MC-9b | 11A | 2.5kW | 4kW | ~9,600~10,470 |
| MC-12b | 13A | 3.5kW | 5.5kW | ~10,100~11,000 |
| MC-18b | 18A | 4.5kW | 7.5kW | ~14,190~16,700 |
| MC-22b | 22A | 5.5kW | 11kW | ~15,700~20,592 |
| MC-32a | 32A | 7.5kW | 15kW | ~28,000~36,432 |

- **사이징**: 설비 명판 정격전류·kW·상·전압 확인 → **AC-3 정격 ≥ 설비 정격전류 × 1.15~1.25**.
- **코일전압 = 제어전원 일치 필수** (국내 배전반 통상 AC220V, 저압제어면 DC24V). 불일치 시 미동작/코일 소손.
- 대안: Schneider TeSys **LC1D09/LC1D12**(국내 정가 확인 불가, LS 동급 대비 높음).
- 국내: 디바이스마트·엘레파츠·나비엠알오(MRO)·미스미·청계천/구로 공구상가.

### B-2. SSR (대안)
- **적합**: 히터/저항성 부하, 고빈도 ON/OFF. **부적합**: 대형 유도성 모터(기동전류 서지로 손상). 모터는 접촉기가 표준.
- 국내 예: **Autonics SR1-1225-N**(25A, 4~30VDC in, 24~240VAC out, ~9,990원~). **방열판 필수, 정격의 절반 이하 디레이팅**.

### B-3. 배선 개념
1. MQTT 릴레이 dry contact(COM–NO)를 제어전원과 접촉기 코일단자 **A1/A2 사이 직렬** 삽입.
2. MQTT ON → 접점 닫힘 → 코일 여자 → 주접점 닫혀 통전. OFF → 코일 소자 → 스프링 복귀로 차단.
3. MQTT 릴레이는 **코일 여자 전류(수백 mA~수 A)만** 감당 → 소형으로 충분.
4. 실무: 접촉기 상단 **차단기(MCCB)** + 모터보호 **열동형 과부하계전기(TOR)** 조합(MMS). 무전원 상태에서 결선·접지·상순 확인 후 통전.

## Part C — Fail-safe & 안전등급 (중요)

> **한 줄 요약**: 프로슈머 WiFi 릴레이는 **인증된 안전장치가 아니다.** Part A 모듈은 "편의 기반 원격 정지"로 쓰고, 인명 위험이 있는 진짜 E-STOP은 **전원 경로에 정격 안전릴레이(하드와이어드)** 를 둔다. 둘은 배타적이 아니라 **병행**이 정석.

### C-1. Fail-safe / NC / De-energize-to-Stop
- 안전상태(정지)는 **신호를 넣어서가 아니라 빼서** 도달 → 단선·컨트롤러 다운·정전이 모두 자동 STOP.
- **정지 입력은 NC(상시 닫힘)** → 선 끊기면 회로 열려 정지(단선=정지=안전). 부하는 접촉기 **NO 주접점**으로 통전 → 코일 드롭아웃 시 개방.
- **접점 용착 대비**: force-guided 안전릴레이/안전접촉기 + 보조접점 feedback 감시. 정지 후 **자동 재기동 금지 → 수동 리셋**.
- WiFi 릴레이를 부하에 직결하지 말고 **안전릴레이 입력 트리거로만** 사용.

### C-2. 안전릴레이 / 등급(ISO 13849 PL, IEC 62061 SIL)
| 제조사 | 예시 | 등급 | 개략가 |
|--------|------|------|--------|
| **Autonics(국산)** ⭐ | **SFC-R212** | **SIL CL3 / PL e / S-Mark** | ~US$150 |
| Pilz | PNOZ X3 | PL e / SIL 3 | ~US$200~430 |
| Schneider Preventa | XPS AF5130 | EN/ISO 13850 | ~US$200~400 |
| Phoenix Contact | PSR-SCP | 듀얼채널 | ~US$200대 |

- **국산 Autonics SFC-R**이 PL e/SIL CL3 + S-Mark → A/S·조달·인증 유리.
- 인명 위험 정지는 통상 **Cat.3~4 / PL d~e (SIL 2~3)** 목표 → 이중채널+자기진단+force-guided 접점 필요. **WiFi 릴레이 단독은 어떤 PL/SIL도 주장 불가.**

| | 최소 동작형(안전등급 아님) | 정식 안전등급형 |
|---|---|---|
| 트리거 | WiFi 릴레이 + MQTT | 하드와이어드 E-STOP(NC) |
| 판단부 | 없음(릴레이 직접) | 안전릴레이(이중채널) |
| 차단부 | 일반 릴레이/접촉기 | 안전접촉기(force-guided)+feedback |
| 등급 | 없음(편의 정지) | Cat.3~4 / PL d~e / SIL 2~3 |
| 용도 | 원격 편의정지·부가알림 | 실제 인명 E-STOP |

### C-3. WiFi vs 유선
- **안전 신호는 유선(하드와이어드)이 정석.** NFPA 79 / IEC 60204 / ISO 13849는 정지 기능이 SW·네트워크 실행에 의존하지 말 것을 요구.
- WiFi/MQTT는 간섭·AP다운·브로커 장애로 드롭 가능 → 최종 안전층에 두지 말 것. Part A에서도 **유선 모델(Shelly Pro 2, ADAM-6266, WISE-7560M PoE)** 우선.

## 최종 권고
1. **MQTT 차단 모듈**: 유선·즉시구축 = **Shelly Pro 2**, 산업조달·페일세이프 = **ADAM-6266(-D)**, 저비용 파일럿 = **Sonoff 4CH Pro R3+Tasmota / USR-M100**. → 발주 전 커스텀 토픽 subscribe 스키마 매뉴얼 확인 필수.
2. **전원 스위칭**: 설비 kW·상에 맞춘 **LS METASOL MC**(코일=제어전원 전압), 히터류만 Autonics SSR+방열판.
3. **안전**: 이 구성 그대로는 "편의 정지". 인명 위험 설비면 **Autonics SFC-R(국산 PL e/SIL CL3)** 등 정격 안전릴레이+안전접촉기를 전원 경로에 하드와이어드 추가. 최종 PLr/SIL은 **유자격 안전엔지니어의 리스크 어세스먼트(ISO 12100)** 로 산정.
