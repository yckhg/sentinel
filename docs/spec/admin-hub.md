# 관리 허브 스펙

> 상태: 살아있는 계약 (living spec) · 독자: 스펙 작성자 / 오케스트레이터
> 역할: 관리 영역의 진입 화면(허브). 전체 관리 기능을 그룹별 목록으로 보여주고
> 각 항목을 대응 서브페이지로 연결한다.
> 상위 맵: [[admin-management-ia-map]] · 접합부(라우팅·게이트): [[admin-routing-navigation-contract]]

## 목적 / 의도

관리 영역이 성격이 다른 다수 기능을 하나의 세로 스크롤 화면에 담아 발견성이 나빴다. 관리 허브는
그 진입점으로서, 전체 관리 기능을 **성격별 그룹으로 묶은 목록**으로 제시하고 각 항목을 딥링크
가능한 전용 서브페이지로 연결한다. 허브 자신은 어떤 관리 데이터도 편집하지 않는다 — 오직
**발견·분류·내비게이션**만 책임진다.

이 단위는 [[admin-management-ia-map]]의 3가지 관통 불변식을 **참조로 상속**한다(여기서 재도출하지
않는다):
1. **행위 보존** — 재구성은 위치·내비게이션만 바꾸고 기능 계약을 바꾸지 않는다.
2. **백엔드·API 무변경** — 허브는 새 서버 계약을 만들지 않는다.
3. **접근 경계 보존(admin 전용)** — 관리 영역 전체(허브 포함)는 admin 전용이다.

## 언어 · 런타임

브라우저 SPA(React). URL이 활성 화면의 단일 진실 소스(SSOT)라는 기존 원칙을 따른다 — 그
메커니즘은 접합부 계약이 소유한다.

## 의존 도구 · 시스템

- **[[admin-routing-navigation-contract]]** (접합부): URL 체계(`/admin`, `/admin/<slug>`),
  드릴다운, 뒤로가기=허브 복귀, admin 게이트, open-redirect 가드를 **소유**한다. 허브는 이
  계약의 인터페이스에만 의존하며 라우팅·게이트 메커니즘을 재정의하지 않는다.
- 각 서브페이지의 내부(그 페이지가 무엇을 편집·표시하는지)는 알지 않는다 — 허브는 항목의
  **라벨 · 그룹 · 대상 canonical path**만 안다.

## 입력

- 인증 컨텍스트: 현재 사용자의 역할(admin / 비-admin / 미인증) — 게이트 판정은 접합부가 소유.
- 진입: 상위 4탭 중 관리 탭 선택, 주소창 직접 `/admin` 진입, 서브페이지에서의 복귀 — 모두
  접합부 계약이 정의한 경로 진입.
- 허브 자체 입력 데이터는 없다(정적 기능 목록).

## 출력 (계약)

- `/admin`에 마운트되어, admin에게 **관리 허브**를 렌더한다(비-admin·미인증은 접합부 게이트가
  처리하며 허브 콘텐츠를 노출하지 않는다).
- 허브는 **4개 그룹 라벨** 아래 **총 10개 기능 항목**을 나열한다. 어떤 기능도 누락되지 않는다.
- 그룹 ↔ 항목 ↔ 대상 canonical path 매핑(계약):

  | 그룹 라벨 | 항목(라벨) | 대상 경로 |
  |---|---|---|
  | **장치** | 장비(센서) 관리 | `/admin/devices` |
  | **장치** | 카메라 관리 | `/admin/cameras` |
  | **장치** | 시스템 상태(장비 상태·예외) | `/admin/health` |
  | **알림·연락** | 연락처 관리 | `/admin/contacts` |
  | **알림·연락** | 비상 신호 시뮬레이션 | `/admin/test-alert` |
  | **알림·연락** | 알림 채널 테스트 발송 | `/admin/notify-test` |
  | **시스템** | 시스템 설정(현장 정보+시스템 설정) | `/admin/system` |
  | **시스템** | 사용자(계정+초대) | `/admin/users` |
  | **저장·CCTV** | 저장소 관리 | `/admin/storage` |
  | **저장·CCTV** | 임시 CCTV 링크 | `/admin/cctv-links` |

- 한 항목을 선택하면 그 항목의 대상 canonical path로 내비게이트한다(드릴다운 메커니즘은 접합부
  소유). 서브페이지로부터의 복귀 도달성도 접합부가 보장한다.

## 핵심 로직 (동작)

- 허브는 위 표의 10개 항목을 **정확히 그 4개 그룹으로 묶어** 제시한다 — 그룹 소속은 위 매핑을
  따른다.
- 각 항목은 자신의 대상 canonical path로 가는 **활성화 가능한 어포던스**(탭·링크)를 가진다.
- 어떤 항목도 다른 항목과 같은 대상 경로를 가리키지 않는다(항목 ↔ slug 1:1).
- 허브는 관리 데이터를 조회·수정하지 않으며, 항목 목록·그룹은 정적이다(사용자 데이터에 따라
  항목이 사라지거나 추가되지 않는다).
- **접근 경계 동등성 참고:** 오늘 일부 관리 섹션은 임의 인증 사용자에게도 보였으나, 설계 맵의
  불변식 ③과 접합부 계약에 따라 허브·전체 관리 영역은 **admin 전용**으로 고정한다. 따라서
  행위 동등성은 이전의 *admin 뷰*를 기준으로 한다.

## 검증 단언 (TDD)

- **A.** admin이 `/admin`에 진입하면 관리 허브가 렌더되고, **4개 그룹 라벨**(장치 · 알림·연락 ·
  시스템 · 저장·CCTV)이 모두 존재한다. — 핵심
- **B.** 허브에 **10개 기능 항목**이 모두 나열되며, 어느 하나도 누락되지 않는다(항목 수 = 10). —
  핵심
- **C.** 각 항목은 [출력 계약]의 표대로 **정확한 그룹**에 속한다(그룹별 항목 집합이 표와 일치). —
  핵심
- **D.** 각 항목의 내비게이션 대상이 표의 대응 canonical `/admin/<slug>`와 일치하며, 서로 다른
  두 항목이 같은 대상을 가리키지 않는다(항목↔slug 1:1, 전 slug가 정확히 한 번 등장). — 핵심
- **E.** 허브의 한 항목을 활성화하면 대응 `/admin/<slug>`로 내비게이트되어 그 서브페이지에
  도달한다(드릴다운 결과 관측). — 핵심
- **F.** 비-admin(비-admin 역할 또는 미인증)이 `/admin`에 진입하면 허브의 관리 항목 목록이
  렌더되지 않고 기존 게이트 동작으로 유도된다. — 핵심
- **G.** 임의 `/admin/<slug>` 서브페이지에서 허브로 복귀하면(뒤로가기/복귀 어포던스) 동일한
  10개 항목 그룹 목록이 다시 관측된다. — 일반
