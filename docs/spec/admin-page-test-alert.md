# 비상 신호 시뮬레이션 서브페이지 (/admin/test-alert) 스펙

> 상태: 살아있는 계약 (living spec) · 독자: 스펙 작성자 / 오케스트레이터
> 역할: 관리 IA 재구성에서 "비상 신호 시뮬레이션" 기능을 담당하는 단일 서브페이지 단위.
> 상위 맵: [[admin-management-ia-map]] · 접합부: [[admin-routing-navigation-contract]]

## 목적 / 의도

관리 영역 재구성 이전 관리 화면 안에 있던 **비상 신호 시뮬레이션** 섹션을,
`/admin/test-alert` 서브페이지로 이전(移轉)해 "한 화면 = 한 관심사" 원칙과 딥링크 가능한 URL을
확보한다. 이 페이지는 **하나의 트리거**로 전체 알림 체인을 [테스트] 접두사로 발동시켜, 운영자가
알림 파이프라인이 실제로 동작하는지 한 번에 검증하게 한다.

이 단위는 상위 맵의 불변식을 **참조로 상속**한다: ① 행위 보존(behavior-preserving) ②
백엔드·API 무변경 ③ admin 전용 접근 경계. 재구성은 위치·내비게이션만 바꾸고 기능 계약을 바꾸지
않는다.

이 페이지는 실제 알람과 동일한 경로로 **전체 알림 체인 하나**를 [테스트] 접두사로 발동한다 —
대상 지정·입력 필드 없이 단일 트리거 하나로 파이프라인 전체를 한 번에 검증한다.

## 언어 · 런타임

브라우저 SPA(React). 화면 진입·복귀·접근 경계는 [[admin-routing-navigation-contract]]가 소유하며,
이 페이지는 그 계약의 인터페이스에만 의존한다.

## 의존 도구 · 시스템

- `POST /api/test-alert` — 재구성으로 **변경되지 않는** 기존 엔드포인트(요청/응답 계약 무변경).
- 관리 영역의 기존 admin 게이트(접합부 계약이 재사용).

## 입력

- 진입: `/admin/test-alert` 딥링크 또는 허브(`/admin`)에서 "비상 신호 시뮬레이션" 항목 선택.
- 인증 컨텍스트: admin / 비-admin / 미인증.
- 사용자 조작: 단일 트리거 버튼 탭. **추가 입력 필드·대상 지정 없음.**

## 출력 (계약)

- `/admin/test-alert`에 마운트되며 admin에게만 렌더된다(접근 경계는 접합부 계약이 소유).
- 화면 구성: 전체 알림 체인을 [테스트] 접두사로 발동한다는 설명 텍스트 + **하나의 경고 성격
  트리거 버튼**("비상 신호 시뮬레이션"). 목록·표는 없다.
- 트리거 시 **바디 없이** `POST /api/test-alert`를 호출한다.
- 발동 성공 시 성공 피드백을 표시하고, 그 피드백은 일정 시간(5초) 뒤 자동으로 사라진다.
- 실패 시 오류 피드백을 표시한다.
- 허브(`/admin`)로 복귀할 수 있다(복귀 수단은 접합부 계약이 소유).

## 핵심 로직 (동작)

- **단일 트리거:** 화면에는 대상 선택·확인 대화상자 없이 하나의 버튼만 있고, 그 버튼이 전체 알림
  체인 발동을 담당한다.
- **체인 설명:** 설명 텍스트는 발동되는 경로가 MQTT → hw-gateway → notifier →
  KakaoTalk/SMS/이메일 전 채널이며, 모든 메시지에 **[테스트] 접두사**가 포함됨을 명시한다.
- **행위 동등:** 트리거→호출→성공/실패 피드백→성공 피드백 자동 소멸 동작이 재구성 이전 "비상
  신호 시뮬레이션" 섹션과 관측상 동등하다.
- **확인 없음:** 발동 전 별도 확인 대화상자를 두지 않는다(이전 동작 보존).

## 검증 단언 (TDD)

- **A.** admin이 `/admin/test-alert`에 진입하면 단일 "비상 신호 시뮬레이션" 트리거 버튼과 체인
  설명 텍스트가 렌더된다(입력 필드·목록 없음). — 핵심
- **B.** 트리거 버튼 탭 시 **바디 없는** `POST /api/test-alert` 요청이 정확히 1회 발생한다. — 핵심
- **C.** 호출 성공 시 성공 피드백이 표시되고, 약 5초 뒤 자동으로 사라진다. — 핵심
- **D.** 호출 실패 시 오류 피드백이 표시된다. — 일반
- **E.** 설명 텍스트가 전체 체인(MQTT→hw-gateway→notifier→KakaoTalk/SMS/이메일)과 [테스트]
  접두사 규칙을 안내한다. — 일반
- **F.** 비-admin·미인증이 `/admin/test-alert`에 진입하면 트리거 UI가 렌더되지 않는다(접합부 E
  스모크 참조; 정본 판정은 접합부 소유). — 핵심
- **G.** `/api/test-alert`의 요청/응답 계약은 재구성 전과 동일하다(바디 없는 POST, admin 전용;
  호출은 동일 admin 인증 헤더를 부착한다). — 핵심
