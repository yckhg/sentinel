# 시스템 설정 서브페이지 (/admin/system) 스펙

> 상태: 살아있는 계약 (living spec) · 독자: 스펙 작성자 / 오케스트레이터
> 역할: 관리 IA 재구성에서 "현장 정보 + 시스템 설정" 두 섹션을 한 서브페이지로 합친 잎(leaf).
> 상위 맵: [[admin-management-ia-map]] · 접합부: [[admin-routing-navigation-contract]]

## 목적 / 의도

관리 영역 재구성에서, 재구성 이전 세로 스크롤 화면의 **현장 정보** 섹션과 **시스템 설정**
섹션을 하나의 서브페이지 `/admin/system`으로 합쳐, 한 화면에서 두 관심사(현장 연락 정보 관리 +
외부 접속 URL 설정)를 함께 다룬다. 위치·내비게이션만 바꾸고 각 섹션의 관측 가능한 동작은
동등하게 보존한다.

이 잎은 상위 맵의 3가지 불변식을 **상속**한다: ① 행위 보존, ② 백엔드·API 무변경,
③ 접근 경계 보존(관리 영역 전체 admin 전용). 라우팅·딥링크·뒤로가기=허브 복귀·admin 게이트·
open-redirect 가드는 이 잎이 정의하지 않고 [[admin-routing-navigation-contract]]에만 의존한다.

이전에 현장 정보 섹션은 인증된 사용자 누구에게나 보였으나, 불변식 ③에 따라 관리 영역 전체가
admin 전용이 되므로 이 페이지도 admin 전용이다. 따라서 현장 정보의 행위 동등성 기준은
**재구성 이전 admin 뷰**(현장 정보 + 시스템 설정을 함께 보던 화면)이다.

## 언어 · 런타임

브라우저 SPA(React). URL이 활성 화면의 단일 진실 소스라는 라우팅 원칙(접합부 계약 소유)을 따른다.

## 의존 도구 · 시스템

- 현장 정보: `GET /api/sites` (목록), `PUT /api/sites/{id}` (수정) — 기존 계약 그대로.
- 시스템 설정: `GET /api/settings` (설정 조회, `site_url` 키), `PUT /api/settings/site_url` — 기존 계약 그대로.
- 접근 경계·라우팅: [[admin-routing-navigation-contract]]가 소유하는 admin 게이트·허브 복귀.

## 입력

- 진입: 허브(`/admin`)에서 "시스템 설정" 항목 선택, 또는 `/admin/system` 직접 진입(딥링크/새로고침).
- 인증 컨텍스트: 현재 사용자 역할(admin / 비-admin / 미인증).
- 현장 정보 편집: 대상 현장의 주소·담당자 이름·담당자 전화번호 입력.
- 시스템 설정 편집: 외부 접속 URL(`site_url`) 값 입력.

## 출력 (계약)

- `/admin/system`에 마운트되며 admin에게만 페이지가 렌더된다. 비-admin·미인증에게는 관리
  콘텐츠가 노출되지 않는다(게이트 동작은 접합부 계약 소유).
- 한 페이지 안에 **현장 정보 블록**과 **시스템 설정 블록**이 함께 존재한다.
- 현장 정보 블록: 현장 목록을 보여주고, 각 현장을 **인라인 편집**할 수 있다(수정 전용 —
  추가·삭제 없음). 저장은 주소·담당자 이름·담당자 전화번호를 `PUT /api/sites/{id}`로 보낸다.
- 시스템 설정 블록: 외부 접속 URL 단일 필드를 편집·저장한다. 저장은 `PUT /api/settings/site_url`.
- 페이지에서 허브(`/admin`)로 복귀할 수 있다(복귀 동작은 접합부 계약 소유).

## 핵심 로직 (동작)

**현장 정보 블록 (site)**
- 목록은 `GET /api/sites`로 채운다. 로딩·오류·빈 목록 상태를 구분해 표시한다.
- 각 현장 카드는 인라인 편집 모드로 전환되어 주소·담당자 이름·담당자 전화번호를 편집한다.
  현장 추가·삭제 어포던스는 없다(수정 전용).
- 저장 검증: **주소 필수**, **담당자 이름 필수**, **담당자 전화번호는 형식
  `^01[016789]-\d{3,4}-\d{4}$` 를 만족**해야 한다. 검증 미충족이면 저장을 거부하고 오류를
  안내한다.
- 검증 통과 시 `PUT /api/sites/{id}` 로 `{address, managerName, managerPhone}` 를 보낸다.
- 편집 취소 시 편집 모드를 벗어나고 변경을 반영하지 않는다.

**시스템 설정 블록 (settings)**
- `GET /api/settings`로 설정을 조회해 `site_url` 키의 현재 값을 단일 필드에 채운다.
- 저장 버튼은 **현재 입력값이 원본 값과 동일(미변경)할 때 비활성**이다.
- 값이 변경된 상태(dirty)에서 **취소**하면 입력을 원본 값으로 되돌린다.
- 저장 성공 시 성공 피드백을 표시한다(피드백은 잠시 후 자동으로 사라진다).
- 저장은 `PUT /api/settings/site_url` 로 값을 보낸다. 값을 비우면 기본값을 사용한다.

**병합·경계**
- 두 블록은 같은 `/admin/system` 화면에 동시에 렌더된다. 한 블록의 편집·저장이 다른 블록의
  상태를 바꾸지 않는다.
- 서버 엔드포인트·요청/응답 계약은 재구성 전과 동일하다(불변식 ②).

## 검증 단언 (TDD)

- **A.** `/admin/system` 직접 진입 시 시스템 설정 서브페이지가 렌더되고, **현장 정보 블록과
  시스템 설정 블록이 한 화면에 함께** 존재한다. — 핵심
- **B.** 비-admin(미인증 또는 비-admin 역할)이 `/admin/system`에 진입하면 관리 콘텐츠가
  렌더되지 않고 접합부 계약의 게이트 동작으로 유도된다. — 핵심
- **C.** 현장 정보 블록은 현장 목록을 보여주고 각 현장을 인라인 편집할 수 있으며, 현장
  추가·삭제 어포던스는 없다(수정 전용). — 핵심
- **D.** 현장 편집 저장 시 **주소 비어 있음** 또는 **담당자 이름 비어 있음**이면 저장이
  거부되고 오류가 안내된다(서버로 전송되지 않는다). — 핵심
- **E.** 현장 편집 저장 시 담당자 전화번호가 `^01[016789]-\d{3,4}-\d{4}$` 를 만족하지 않으면
  저장이 거부되고, 만족하면 `PUT /api/sites/{id}` 로 `{address, managerName, managerPhone}` 가
  전송된다. — 핵심
- **F.** 시스템 설정 블록은 `site_url`의 현재 값을 편집할 수 있고, 저장 시
  `PUT /api/settings/site_url` 로 값이 전송된다. — 핵심
- **G.** 시스템 설정 저장 버튼은 입력값이 원본과 동일할 때 비활성이고, 값이 변경되면
  활성화된다. — 핵심
- **H.** 시스템 설정에서 값을 변경한 뒤 취소하면 입력이 원본 값으로 복원된다. — 핵심
- **I.** 시스템 설정 저장 성공 시 성공 피드백이 표시되고, 이후 자동으로 사라진다. — 일반
- **J.** 이 서브페이지에서 허브(`/admin`)로 복귀할 수 있다(접합부 계약 동작). — 일반
- **K.** 현장 정보·시스템 설정이 사용하는 서버 엔드포인트(`/api/sites`, `/api/settings`,
  `/api/settings/site_url`)의 요청/응답 계약은 재구성 전과 동일하다. — 핵심
