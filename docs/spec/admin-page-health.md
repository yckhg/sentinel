# 관리 서브페이지 — 장비 상태·예외 (/admin/health) 스펙

> 상태: 살아있는 계약 (living spec) · 독자: 스펙 작성자 / 오케스트레이터
> 역할: 관리 IA 재구성에서 "장비 상태·예외" 관심사를 canonical 경로 `/admin/health`의
> 단일 서브페이지로 분리한 리프(leaf) 계약.

## 목적 / 의도

관리 영역 재구성에서, 재구성 이전 세로 스크롤 관리 화면에 포함돼 있던 **건강/예외 패널
("시스템 상태")** 을 딥링크 가능한 독립 서브페이지 `/admin/health`로 이관한다. 이 페이지는
운영자가 보는 **현재-상태 요약 창**을 렌더하는 **읽기 전용 표시 화면**이다 — 서비스 상태·장비
요약 카운트·예외(이상/오프라인) 장비 목록·특정 장비 현재 상태 검색·특정 장비 상태 전이 이력
드릴다운·카메라 연결 상태를 관측한다. 어떤 쓰기/CRUD도 하지 않는다.

이 페이지가 소유하는 계약은 두 갈래다.

1. **행위 보존 (상속):** 이 재구성은 순수 정보구조(IA)·표현 계층 변경이다. 이 페이지는
   [[admin-management-ia-map]]의 세 불변식 — ① 행위 보존 ② 백엔드·API 무변경 ③ 접근 경계
   보존(admin 전용) — 을 **참조로 상속**한다. 이 페이지가 표시하는 현재-상태 요약 창의 관측
   가능한 동작에 대한 **정본 정의(canonical definition)** 는 [[system-status-aggregate]]
   (시스템 상태 집계 스펙)가 소유하며, 본 리프는 그 정의와 **정합**하게, 재구성 이전 "시스템
   상태" 패널이 admin 화면 안에서 보이던 동작을 `/admin/health`에서 **동등하게** 보존한다.
2. **#112 레이아웃 개선 (본 리프의 배타적 추가 소유):** 예외 목록이 다수일 때 단일 좁은
   컬럼으로 나열돼 발견성이 나쁘고 과도한 세로 스크롤이 필요하다는 문제(**#112**)의 레이아웃
   개선을, **오직 이 페이지 스펙만이** 명시적으로 소유한다(다른 어떤 리프도 이 단언을 갖지
   않는다). 이 개선은 위 행위 보존 불변식과 **별개**이며(동작 동등성이 아니라 표현 개선),
   #112를 해소하는 load-bearing 계약이다.

URL 체계·딥링크·뒤로가기=허브 복귀·admin 게이트·미정의 경로 폴백·open-redirect 가드는 본
리프가 재정의하지 않고 **[[admin-routing-navigation-contract]]** (라우팅·내비게이션 접합부
계약)에만 의존한다.

## 언어 · 런타임

모바일 우선 React SPA(정적 빌드, nginx 서빙). URL(경로)이 활성 화면의 단일 진실 소스라는 기존
라우팅 원칙을 따른다. 백엔드 응답 shape에만 의존하며 백엔드 내부는 알지 않는다. (식별용이며
빌드 지시가 아니다.)

## 의존 도구 · 시스템

- **[[admin-routing-navigation-contract]]** — canonical 경로 `/admin/health` 마운트, 딥링크,
  허브 복귀, admin 전용 접근 경계, 미정의 경로 폴백을 소유하는 횡단 계약. 본 페이지는 이
  계약의 인터페이스에만 기대며 그 내부를 알지 않는다.
- **[[system-status-aggregate]]** — 이 페이지가 표시하는 요약 창의 집계·검색·드릴다운·카메라
  연결 상태 동작에 대한 정본 행위 계약. 본 페이지의 행위 동등성 단언은 이 스펙의 정의를 참조한다.
- 백엔드 읽기 접면(무변경, 참조): 요약 집계 조회, 카메라 연결 상태 조회, 장비 단건 현재-상태
  검색 조회, 장비 상태 전이 이력 조회. 접면의 계약 SSOT는 위 두 문서와 접면 SSOT가 소유하며
  본 리프는 이를 **바꾸지 않는다**.

## 입력

- 경로 진입: 허브에서의 드릴다운, 주소창 직접 진입(딥링크), 새로고침 — 모두 접합부 계약 소관.
- 인증 컨텍스트: 현재 사용자의 역할(admin / 비-admin / 미인증).
- 운영자 상호작용: 장비 검색어 입력(`deviceId` 또는 `siteId:deviceId`), 예외 목록 항목 선택
  (이력 드릴다운 요청).
- 백엔드 읽기 응답: 요약 집계, 카메라 연결 상태, 장비 단건 조회, 장비 이력.

## 출력 (계약)

`/admin/health`에 admin으로 진입하면 재구성 이전 "시스템 상태" 패널과 **관측 동등한** 읽기 전용
현재-상태 요약 창이 렌더된다. 화면에는 아래가 존재하며 각각의 정본 정의는
[[system-status-aggregate]]를 따른다:

- 장비 **요약 카운트**(정상 / 이상 / 오프라인) 표시.
- 감시 서비스의 **고정 집합 전체** 목록과 각 서비스의 정상/이상 상태.
- **예외(이상·오프라인) 장비 목록** — 예외 장비만 개별 행으로. 정상(healthy) 장비는 목록에
  나열되지 않는다. 예외가 상한을 초과하면 **오버플로 표식**(남은 예외 수)이 표시된다.
- **장비 검색** 입력과 그 결과(특정 장비의 현재 상태 — 정상 장비도 조회 가능, 미등록/삭제 장비는
  부재로 처리).
- 예외 목록의 한 항목을 선택하면 그 장비의 **상태 전이 이력**(online/offline 전이)이 드릴다운
  으로 표시된다.
- **카메라 연결 상태**(연결됨 / 연결 끊김) 표시(집계 밖 별도 관측).
- 요약 창은 **주기적으로 갱신**된다(현재-상태 폴링).

읽기 전용 계약: 이 페이지는 어떤 백엔드 상태도 변경하지 않으며, 서버 접면·요청/응답 계약을
바꾸지 않는다(불변식 ②). 모든 읽기 접면 호출은 동일 admin 인증 헤더를 부착한다.

## 핵심 로직 (동작)

- **행위 동등성:** 위 관측 항목의 동작 규칙(정상=카운트·예외만 개별, 경계성, 검색은 나열이 아닌
  단건 조회, 이력은 드릴다운 전용·타 장비 전이 미포함, 서비스 목록 항상 완전, 카메라 연결은
  집계 밖 별도 관측)은 [[system-status-aggregate]]가 정의한 그대로다. 본 페이지는 이를
  재구성 이후 `/admin/health`에서 **동등하게** 보존하며 새 동작을 도입하지 않는다.
- **접근 경계(admin 전용):** 재구성 이후 `/admin` 영역 전체는 admin 전용이다(불변식 ③). 따라서
  이 페이지의 관측 동등성 기준은 **admin 사용자가 보는 화면**이며, 비-admin·미인증 접근에서
  관리 콘텐츠가 노출되지 않게 하는 게이트 동작은 [[admin-routing-navigation-contract]]가
  소유한다(본 리프는 재정의하지 않는다).
- **#112 예외 목록 레이아웃(본 리프 소유, 행위 보존과 별개의 개선):** 예외 장비 목록은 항목이
  **다수일 때 하나의 좁은 세로 컬럼에 갇히지 않는다**. 다수 예외를 발견성 높은 형태(반응형
  다열/그리드 배치 또는 동등한, 세로 스크롤을 과도하게 요구하지 않는 배치)로 제시한다. 소수·통상
  건수의 예외에서도 배치는 정상적으로 렌더된다. 이 배치 개선은 어떤 항목이 목록에 담기는가(예외
  집합·정상 미나열·오버플로 규칙)를 바꾸지 않는다 — 오직 담긴 항목의 **표현**만 개선한다.

## 검증 단언 (TDD)

관측 대상은 admin으로 `/admin/health`에 진입한 화면이다. 각 항목의 정본 판정 기준은
[[system-status-aggregate]]의 대응 단언을 참조한다. 아래는 이 서브페이지에서의 관측 동등성·
접근 경계·#112 배치를 OK/NOK로 판정한다.

- **A. 요약 카운트 표시 (핵심).** 정상/이상/오프라인 각 범주의 장비가 있을 때, 화면에 세 범주의
  요약 카운트가 각각 표시되고 그 수가 집계 응답과 일치한다.
- **B. 서비스 목록 항상 완전 (핵심).** 감시 서비스 고정 집합 전체가 목록에 표시되고 각 항목이
  정상/이상 상태를 가진다. 모든 서비스가 정상이어도 목록은 축약되지 않으며, 한 서비스가 이상
  이면 그 서비스만 이상으로, 나머지는 여전히 목록에 존재한다.
- **C. 예외 목록 = 이상/오프라인만, 정상 미나열 (핵심).** 예외 장비 목록에는 이상·오프라인 장비
  만 개별 행으로 등장하고, 정상(healthy) 장비의 식별자는 하나도 등장하지 않는다. 예외가 0이면
  "예외 없음"에 해당하는 빈 상태가 표시된다.
- **D. 예외 항목 → 이력 드릴다운 (핵심).** 예외 목록의 한 항목을 선택하면 그 장비의 상태 전이
  이력(online/offline 전이) 최근 N건이 표시되고, 다른 장비의 전이는 하나도 포함되지 않는다.
  요약 창의 기본 로드에는 어떤 전이 이력도 실려 있지 않다.
- **E. 장비 검색 = 단건 현재-상태 (핵심).** 운영자가 `siteId:deviceId`(또는 deviceId)로 특정
  장비를 지목하면 그 장비의 현재 상태(생존 여부/마지막 관측 경과/이상 여부)가 조회돼 표시된다 —
  정상으로 분류된 장비도 검색으로 조회된다(예외 목록엔 없더라도). 미등록/삭제 장비 식별자로
  검색하면 부재(찾을 수 없음)로 처리되고 화면이 깨지지 않는다.
- **F. 예외 오버플로 표식 (일반).** 예외 수가 상한을 초과하면 화면에 개별 행은 상한까지만 그려
  지고, 남은 예외 수를 대표하는 오버플로 표식이 표시된다. 상한 이하면 오버플로 표식은 없다.
- **G. 카메라 연결 상태 관측 (핵심).** 연결된 카메라는 화면에서 연결됨으로, 스트리밍이 끊긴
  카메라는 연결 끊김으로 관측된다(집계 밖 별도 조회 — 정본은 [[system-status-aggregate]] 단언 G).
- **H. 현재-상태 주기 갱신 (일반).** 화면이 열려 있는 동안 요약 창은 주기적으로 재조회되어
  최신 현재-상태를 반영한다(정지된 1회 스냅샷이 아니다).
- **I. admin 전용 렌더 · admin 뷰 동등 (핵심).** admin으로 `/admin/health`에 (딥링크 포함)
  진입하면 위 A–H를 갖춘 요약 창이 렌더되며, 이는 재구성 이전 admin 화면 안 "시스템 상태"
  패널과 관측 동등하다. 비-admin·미인증 접근에서 관리 콘텐츠가 노출되지 않는 게이트 동작은
  [[admin-routing-navigation-contract]]가 소유하므로 본 단언은 그 계약을 참조하며 여기서
  재검증하지 않는다(접합부 E 스모크 참조; 정본 판정은 접합부 소유).
- **J. #112 예외 목록 다열 배치 (일반 — 개선, 행위 보존 아님 · #112 load-bearing).**
  **데스크톱 폭(뷰포트 너비 ≥1024px)** 에서 예외 항목 **N=8 픽스처**를 렌더하면, 렌더된 예외
  항목들의 배치가 **2열 이상**이다 — 항목 bounding-box의 x-좌표가 **둘 이상의 클러스터**로 나뉜다
  (단일 컬럼에 갇히지 않음). **좁은 폭(모바일)에서는 1열이 정상**이며(모바일 우선 앱에서의 정상
  예외), 소수(N이 작음) 예외에서도 배치는 정상적으로 렌더된다. Playwright가 예외 항목들의
  x-좌표 클러스터 수를 관측해 OK/NOK로 판정한다. **이 단언은 [[admin-management-ia-map]]의 행위
  보존 불변식과 별개의 개선이며 #112를 해소하는 load-bearing 계약으로, 오직 이 리프만 소유한다.**

## 검증 스킵 선언 (선택)

- (없음)
