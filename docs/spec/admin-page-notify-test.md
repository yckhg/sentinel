# 알림 채널 테스트 발송 서브페이지 (/admin/notify-test) 스펙

> 상태: 살아있는 계약 (living spec) · 독자: 스펙 작성자 / 오케스트레이터
> 역할: 관리(/admin) IA 재구성에서 "알림 채널 테스트 발송" 기능을 자기 canonical
> 경로(/admin/notify-test)의 서브페이지로 분리한 잎(leaf) 단위.

## 목적 / 의도

관리 세로 스크롤 화면 안에 있던 **알림 채널 테스트 발송** 관심사를 딥링크 가능한
독립 서브페이지 `/admin/notify-test`로 옮긴다. 이는 **순수 IA·표현 계층 이동**이며,
관리자가 이메일·SMS 채널이 실제로 동작하는지 채널마다 한 번씩 단일 대상으로 확인하는
**관측 가능한 동작을 동등하게 보존**한다. 위치·내비게이션만 바뀌고 기능 계약은 바뀌지 않는다.

상속하는 불변식(참조로만 상속, 여기서 재정의하지 않음):

1. **행위 보존** — 재구성 이전 "알림 채널 테스트 발송" 섹션의 관측 동작을 동등 보존한다.
2. **백엔드·API 무변경** — `GET /api/notifications/channels`·`POST /api/notifications/test`
   요청/응답 계약은 바뀌지 않는다.
3. **접근 경계 보존** — 이 서브페이지는 기존과 동일한 **admin 전용** 게이트를 유지한다.

## 참조 계약 (소유·경계)

- **[[admin-routing-navigation-contract]]** — URL 체계·딥링크·뒤로가기=허브 복귀·admin
  게이트·미정의 경로 폴백을 소유한다. 이 잎은 그 인터페이스에만 의존하며, 라우팅·게이트를
  스스로 정의하지 않는다.
- **[[notification-test-send]]** — 채널별 테스트 발송의 **정본 행위 계약**(채널 독립·요청
  시점 config 판정·`not_configured` 무크래시·팬아웃 없음·KakaoTalk 범위 밖·채널 조회 실패 시
  거짓 미설정 강등 금지·outcome 3종)을 소유한다. 이 잎은 그 정의를 참조하며 재정의하지 않고,
  그 계약을 이 경로의 서브페이지 표현으로 노출한다는 점만 고정한다.
- **[[admin-management-ia-map]]** — 이 잎이 속한 상위 설계 맵. 이 단위의 한 줄 책임은 맵의
  "알림 채널 테스트 발송 → /admin/notify-test" 항목이다.

## 언어 · 런타임

브라우저 SPA(React) 관리자 UI. URL(경로)이 활성 화면의 SSOT라는 라우팅 원칙을 따른다(접합부
계약 소유). (식별용 — 구현 지시가 아니다.)

## 의존 도구 · 시스템

- `GET /api/notifications/channels` — in-scope 채널(`email`, `sms`) 각각의
  `{usable, reason}`을 읽는 admin 표면. 계약 정본은 [[notification-test-send]].
- `POST /api/notifications/test {channel, target}` — 지정 채널·단일 대상 단건 동기 발송,
  `outcome ∈ {sent, failed, not_configured}` 반환. 계약 정본은 [[notification-test-send]].
- 라우팅·admin 게이트 — [[admin-routing-navigation-contract]].

## 입력

- 경로 진입: 허브에서 항목 선택(드릴다운), `/admin/notify-test` 직접 진입(딥링크), 새로고침,
  브라우저 뒤로/앞으로. (진입·복귀 규율은 접합부 계약 소유.)
- 인증 컨텍스트: 현재 사용자 역할(admin / 비-admin / 미인증).
- 관리자가 그 자리에서 입력한 **이메일 테스트 대상 주소 1개**, **SMS 테스트 대상 전화번호 1개**.

## 출력 (계약)

- 이 서브페이지는 canonical 경로 `/admin/notify-test`에 마운트되고, **admin에게만** 렌더되며,
  허브(`/admin`)로 돌아갈 수 있다(접합부 계약).
- 화면은 채널별(이메일·SMS)로 **단일 대상 입력 + 단건 테스트 전송** 어포던스를 제공한다.
- **채널 사용가능 표시·비활성:** 채널 조회 결과 해당 채널 `usable===false`이면 그 채널의 전송
  버튼은 비활성화되고 "미설정 — 테스트를 보낼 수 없습니다" 취지의 안내를 노출한다. 성공한 것처럼
  보이게 하지 않는다.
- **거짓 미설정 강등 금지:** 채널 조회 자체가 실패해 채널 상태를 알 수 없는 동안에는 전송
  버튼을 **거짓으로 비활성화하지 않는다**(미설정과 조회불가를 구분; 정본은 [[notification-test-send]]).
- **outcome 표시(3종):** 테스트 결과를 `sent`(성공 표시)·`not_configured`("채널이 미설정
  상태입니다" 취지의 오류 표시)·`failed`("발송 실패: {사유}" 취지의 오류 표시)로 사용자에게
  구분 표시한다. HTTP 계층 거절(입력 오류·레이트리밋 초과·하위 서비스 통신 실패)도 사용자에게
  구분 가능한 오류로 표시하며 거짓 성공으로 보이지 않는다.
- **API 무변경:** 위 두 엔드포인트의 요청/응답 계약은 이 이동으로 바뀌지 않는다.

## 핵심 로직 (동작)

- 이 서브페이지는 [[notification-test-send]]의 채널별 테스트 발송 정본 계약을 `/admin/notify-test`
  경로의 관리자 화면 표현으로 노출한다: 채널 독립·요청 시점 판정·단일 대상 격리·정직한 성패
  보고·KakaoTalk 범위 밖은 모두 그 정본에서 상속하며 여기서 재정의하지 않는다.
- 진입/딥링크/복귀/게이트/미정의 폴백은 [[admin-routing-navigation-contract]]에서 상속한다.
- 이 잎이 고정하는 것은 그 두 계약의 **접합**뿐이다 — 즉 "이 기능이 이 canonical 경로에서,
  admin에게만, 이전 섹션과 동등한 행위로 나타난다".

## 검증 단언 (TDD)

- **A (핵심). 마운트 경로**: `/admin/notify-test` 직접 진입(딥링크) 시 허브가 아니라 알림 채널
  테스트 발송 서브페이지(이메일·SMS 채널별 테스트 전송 어포던스 포함)가 렌더된다. OK: 해당
  서브페이지 렌더. NOK: 허브·타 서브페이지·빈 화면·크래시. (접합부 B 스모크 참조; 정본 판정은
  접합부 소유.)
- **B (핵심). admin 전용**: admin은 이 서브페이지에 도달해 콘텐츠를 본다. 비-admin·미인증이
  `/admin/notify-test`에 진입하면 관리 콘텐츠가 렌더되지 않고 기존 게이트 동작으로 유도된다
  (접합부 E 스모크 참조; 정본 판정은 접합부 소유). OK: admin만 콘텐츠 노출. NOK: 비-admin·미인증에게 콘텐츠 노출.
- **C (핵심). 채널 사용가능 표시·비활성**: 채널 상태에서 한 채널이 `usable===false`이면 그
  채널의 전송 버튼이 비활성이고 미설정 안내가 표시된다. OK: 미설정 채널 버튼 비활성 + 안내.
  NOK: 미설정 채널 버튼이 활성이거나 안내 없음.
- **D (핵심). 단일 대상 단건 발송**: 각 채널의 전송은 관리자가 입력한 **단일 대상**에게
  **1건**만 시도하고 그 성패를 동기로 표시한다. 등록 비상연락처 팬아웃·타 채널 부수 발동이
  이 화면 조작으로 발생하지 않는다(정본 [[notification-test-send]]). OK: 지정 대상 단건.
  NOK: 팬아웃·교차 채널 발동.
- **E (핵심). outcome 3종 구분 표시**: 응답 `outcome`이 `sent`이면 성공 표시, `not_configured`
  이면 미설정 취지의 오류 표시, `failed`이면 사유를 동반한 실패 취지의 오류 표시로 각각 구분
  렌더된다. OK: 3종이 서로 구분되는 성공/오류로 표시. NOK: `failed`/`not_configured`가 성공처럼
  보이거나 3종이 구분되지 않음.
- **F (핵심). 거짓 미설정 강등 금지(조회 실패)**: 채널 조회가 실패해 채널 상태를 알 수 없는
  동안에는 전송 버튼이 거짓으로 비활성화되지 않는다(미설정과 조회불가 구분). OK: 조회 실패 시
  버튼 비-강제비활성. NOK: 조회 실패를 미설정으로 강등해 버튼을 거짓 비활성.
- **G (핵심). 허브 복귀**: 이 서브페이지에서 뒤로가기 또는 페이지 내 뒤로 어포던스로
  허브(`/admin`)로 복귀한다(접합부 C 스모크 참조; 정본 판정은 접합부 소유). OK: 허브 복귀. NOK: 복귀 실패·타 화면.
- **H (일반). API 무변경**: 이 서브페이지는 `GET /api/notifications/channels`·
  `POST /api/notifications/test {channel, target}`를 이전과 동일한 요청/응답 계약으로 호출한다
  (각 호출은 동일 admin 인증 헤더를 부착한다). OK: 엔드포인트·페이로드·응답 해석 동일. NOK: 경로·요청/응답 계약 변경.

## 검증 스킵 선언 (선택)

- 실제 채널 발송 성패 분기(`sent`/`failed`)의 non-vacuous 판정에 필요한 공급자 픽스처(mock
  SMTP·SMS·실패 주입)는 [[notification-test-send]]가 소유·선언한다. 이 잎의 단언은 서브페이지
  **표현·접합**(경로·게이트·표시·비활성·복귀·API 무변경) 수준에서 판정하며, 백엔드 발송 성패
  분기의 픽스처 요건은 정본 스펙의 스킵 선언을 따른다. · 중요도: 일반 · 해소 조건: 정본 스펙의
  공급자 픽스처 확보 시 그쪽에서 판정.
