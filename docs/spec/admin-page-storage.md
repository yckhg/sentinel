# 저장소 관리 서브페이지 (/admin/storage) 스펙

> 상태: 살아있는 계약 (living spec) · 독자: 스펙 작성자 / 오케스트레이터
> 역할: 관리 IA 재구성에서 "저장소 관리" 단위 — 저장소 사용량 + 아카이브 관리를
> `/admin/storage` 서브페이지로 분리한 얇은 잎(leaf) 계약.

## 목적 / 의도

이전 관리 화면의 **저장소 관리** 섹션(저장소 사용량 + 아카이브(보관 영상) 관리)을
`/admin/storage` 단독 서브페이지로 옮긴다. 이는 순수 정보구조(IA)·표현 계층 이동이며,
관측 가능한 동작을 그대로 보존한다.

이 잎은 [[admin-management-ia-map]]의 세 불변식을 **참조로 상속**한다:
① 행위 보존(behavior-preserving) ② 백엔드·API 무변경 ③ admin 전용 접근 경계 보존.
새로운 기능·상태·엔드포인트를 도입하지 않는다.

## 언어 · 런타임

브라우저 SPA(React). 활성 화면의 단일 진실 소스는 URL이다(식별용 — 빌드/구현 지시 아님).

## 의존 도구 · 시스템

- 라우팅·딥링크·뒤로가기(=허브 복귀)·admin 게이트: **[[admin-routing-navigation-contract]]**
  가 소유한다. 이 잎은 그 인터페이스에만 의존하며 라우팅·게이트를 재정의하지 않는다.
- 아카이브 상태 수명주기·다운로드 게이트·완료 시각·실패 노출의 **정본 정의**:
  **[[archive-download-ux]]**(status enum, 다운로드=`completed`에만 게이트, 진행 "준비됨"
  표면화, `failed` 사유 노출). 이 잎은 그 계약과 정합을 유지하며 저장소 관리 화면 표현만 다룬다.
- REST 접면(무변경): `GET /api/storage`, `GET /api/archives`,
  `DELETE /api/archives/{id}`, `DELETE /api/archives/incident/{incidentId}`,
  `GET /api/archives/{id}/download`.

## 입력

- admin 인증 컨텍스트.
- `/admin/storage` 경로 진입(허브 드릴다운 · 딥링크 · 새로고침 · 뒤로/앞으로).
- 저장소 사용량 조회 결과(`/api/storage`).
- 아카이브 목록 조회 결과(`/api/archives`) — 항목별 `status` / 크기 / incident 귀속 / 실패
  사유를 담은 표현(정본은 [[archive-download-ux]] 접합부).

## 출력 (계약)

- 이 화면은 `/admin/storage` canonical 경로에 마운트되고 **admin에게만** 렌더된다(게이트는
  접합부 계약 소유). 허브(`/admin`)로 돌아갈 수 있다.
- **저장소 사용량**:
  - 디스크 사용량 바 — 사용률(%)을 표시하고, 사용률 **≥80% 시 경고 표시**를 함께 노출한다.
  - 사용량 세부(전체 / 사용 / 가용) 및 **구성 분해(녹화 데이터 · 보관 영상 · 합계)** 를 표시한다.
- **아카이브(보관 영상) 목록** — **incident 단위로 그룹화**하여 표시한다. 각 그룹은:
  - 그룹 롤업 상태 배지(**완료 / 처리중 / 실패**)를 노출한다.
  - 그룹에 속한 카메라(스트림) 수와 그룹 합계 용량을 표시한다.
  - 그룹 내 **카메라별 행**을 표시하고, 각 행은 자신의 상태를 표면화한다:
    - `completed` 행: 크기를 표시하고 **활성 다운로드 액션**을 제공한다(다운로드는
      `completed`에만 게이트 — [[archive-download-ux]] 정본).
    - 미완료(처리중/대기 등) 행: 진행 상태를 표시하고 활성 다운로드를 제공하지 않는다.
    - `failed` 행: **실패 사유를 표면화**하고 다운로드를 제공하지 않는다.
- **삭제**:
  - 카메라별 아카이브 **단건 삭제** — 확인 후 수행하며, 성공 시 해당 항목이 목록에서 사라진다.
  - incident **그룹 전체 삭제**(해당 사건의 모든 카메라 아카이브) — 확인 후 수행하며, 성공 시
    그 그룹 항목들이 목록에서 사라진다.
- 아카이브가 하나도 없으면 빈 상태 안내를 보이고, 조회 중/조회 실패 시 각각 로딩·오류 상태를 보인다.

## 핵심 로직 (동작)

- 화면 진입·상태 표시·삭제·다운로드는 이전 저장소 관리 섹션과 **관측적으로 동등**하다 —
  위치·내비게이션만 바뀌고 기능 계약은 바뀌지 않는다.
- 다운로드 affordance는 오직 `completed`에만 존재한다(미완료/`failed`에서는 활성 다운로드 없음).
  다운로드·상태의 정본 정의는 [[archive-download-ux]]가 소유하며, 이 화면은 그것을 소비·표면화한다.
- 단건 삭제·그룹 전체 삭제는 모두 **확인 단계**를 거친 뒤에만 실제 삭제를 호출한다.
- 서버 엔드포인트·요청/응답 계약을 바꾸지 않는다 — 기존과 동일한 API를 동일하게 호출한다.

## 검증 단언 (TDD)

- **S1 (핵심)** — `/admin/storage` 진입 시(딥링크 포함) 저장소 관리 화면이 렌더되고, admin이
  아니면 렌더되지 않는다. 허브로 복귀할 수 있다. (게이트·라우팅·복귀 세부는 접합부 B/E/C
  스모크 참조이며 정본 판정은 [[admin-routing-navigation-contract]]가 소유 — 이 잎은 마운트
  지점만 단언한다.)
- **S2 (핵심)** — 디스크 사용률이 표시되고, 사용률 **≥80%** 이면 경고 표시가 함께 노출된다
  (80% 미만이면 경고가 없다).
- **S3 (핵심)** — 저장소 구성 분해(녹화 데이터 · 보관 영상 · 합계)가 표시된다.
- **S4 (핵심)** — 아카이브 목록이 **incident 단위로 그룹화**되고, 각 그룹이 롤업 상태 배지
  (완료/처리중/실패) · 카메라 수 · 그룹 합계 용량을 노출한다.
- **S5 (핵심)** — 카메라별 행에서 활성 다운로드 액션은 `status == "completed"`인 행에만
  노출되고, 미완료/`failed` 행에는 활성 다운로드가 없다([[archive-download-ux]] 게이트와 정합).
- **S6 (일반)** — `status == "failed"`인 카메라 행은 실패 사유를 표면화한다(사유가 조용히 사라지지 않는다).
- **S7 (핵심)** — 카메라별 아카이브 단건 삭제는 **확인 단계**를 거친 뒤 `DELETE /api/archives/{id}`
  로만 수행되며, 성공 시 그 항목이 목록에서 사라진다. 삭제 요청이 실패하면 오류를 표면화하고
  성공으로 오인하지 않는다(#103 무음 삼킴 방지).
- **S8 (핵심)** — incident 그룹 전체 삭제는 **확인 단계**를 거친 뒤
  `DELETE /api/archives/incident/{incidentId}`로만 수행되며, 성공 시 그 그룹 항목들이
  목록에서 사라진다. 삭제 요청이 실패하면 오류를 표면화하고 성공으로 오인하지 않는다(#103
  무음 삼킴 방지).
- **S9 (핵심)** — `/api/storage`·`/api/archives`(및 삭제·다운로드 접면)의 요청/응답 계약은
  재구성 후에도 변경되지 않는다(기존 API를 동일하게 호출; 각 호출은 동일 admin 인증 헤더를 부착한다).
- **S10 (일반)** — 아카이브가 0개이면 빈 상태 안내가 표시되고, 조회 중/조회 실패 시 각각
  로딩·오류 상태가 표시된다.

## 검증 스킵 선언 (선택)

- (없음) — 다운로드 실제 미디어 게이팅·완료 시각·`failed` 실시간 표면화의 판정은
  [[archive-download-ux]]의 단위 A/B 단언(및 그 스킵 선언)이 소유한다. 이 잎은 그 정본을
  참조·소비하며 중복 판정하지 않는다.
