# 장비(센서) 관리 서브페이지 — /admin/devices 스펙

> 상태: 살아있는 계약 (living spec) · 독자: 스펙 작성자 / 오케스트레이터
> 소속: [[admin-management-ia-map]]의 "장비(센서) 관리 → /admin/devices" 단위(잎).
> 이 잎은 관리 허브 아래 한 서브페이지의 **표현·위치 계약**만 소유한다. 센서 장치의
> 등록·삭제·재활성·재출현의 **행위 계약 자체**는 [[sensor-device-lifecycle]]가 소유하며,
> 라우팅·진입·복귀·접근 경계는 [[admin-routing-navigation-contract]](접합부 계약)가 소유한다.
> 본 잎은 그 두 계약의 인터페이스에만 의존하고, 그것들을 재정의하지 않는다.

## 목적 / 의도

재구성 이전 관리 화면의 세로 스크롤에 섞여 있던 **"장비(센서) 관리" 섹션**을, 딥링크
가능한 독립 서브페이지 `/admin/devices`로 분리한다. 이는 **순수 정보구조(IA)·표현 계층
변경**이며, 상위 설계 맵의 3대 불변식을 **참조로 상속**한다:

1. **행위 보존** — 이 페이지는 이전 "장비(센서) 관리" 섹션의 관측 가능한 동작(목록·추가·
   별칭 편집·삭제·재활성·상태 표시·재출현 안내)을 **동등하게** 보장한다. 위치·내비게이션만
   바뀌고 기능 계약은 바뀌지 않는다.
2. **백엔드·API 무변경** — 이 페이지가 호출하는 서버 엔드포인트·요청/응답 계약은 기존과
   동일하다. 새 엔드포인트를 도입하지 않는다.
3. **접근 경계 보존** — 이 페이지는 기존 admin 전용 게이트를 유지한다(접합부 계약 소유).

이전 이 섹션은 임의 인증 사용자에게 보였으나, 불변식 ③에 따라 admin 전용으로 고정된다. 행위
동등성 기준선은 **이전 admin 뷰**이며, 비-admin 접근 상실은 의도된 강화다.

## 언어 · 런타임

브라우저 SPA(React + TypeScript, Vite). URL이 활성 화면의 단일 진실 소스라는 기존 라우팅
원칙을 따른다(접합부 계약 소유).

## 의존 도구 · 시스템

- **접합부 계약** [[admin-routing-navigation-contract]] — canonical 경로 `/admin/devices`,
  딥링크·뒤로가기=허브 복귀·admin 게이트·미정의 경로 폴백을 소유. 본 잎은 이 계약의 결과만
  기대한다.
- **행위 계약** [[sensor-device-lifecycle]] — 센서 장치의 명시 등록/재활성(계약 1), 스티키
  삭제·재출현 경보(계약 2), 관리와 안전기능의 독립(계약 3), 장치 관리 UI(계약 4)의 규범
  정의. 아래 동등성 단언은 이 계약을 **참조**하며 생명주기를 재규정하지 않는다.
- **기존 API 계약(무변경)**:
  - `GET /api/devices` — 미삭제(활성) 장치 목록.
  - `GET /api/devices/all` — 삭제 포함 전체 목록.
  - `GET /api/settings` — 런타임 설정. 상태 판정용 `health.sensor_alive_threshold_sec`(미설정
    시 60초 폴백)을 읽는다.
  - `POST /api/devices` — 추가·재활성 공용(생성-또는-재활성, [[sensor-device-lifecycle]] 계약 1).
  - `PATCH /api/devices/{id}` — 별칭 수정.
  - `DELETE /api/devices/{id}` — 소프트/스티키 삭제(204).
- **WebSocket `/ws`** — 재출현 통지 `device_reappeared`([[sensor-device-lifecycle]] 계약 2).

## 입력

- 경로 진입: 관리 허브에서 "장비(센서) 관리" 항목 선택, 또는 `/admin/devices` 딥링크·새로고침
  (접합부 계약이 진입을 소유).
- 인증 컨텍스트: admin / 비-admin / 미인증.
- 운영자 조작: 장치 추가 폼 입력, 별칭 인라인 편집, 삭제, 재활성, 재출현 배너 조작.
- 서버 이벤트: WS `device_reappeared` 수신.

## 출력 (계약)

- 이 서브페이지 화면이 canonical 경로 `/admin/devices`에 마운트되어, 이전 "장비(센서) 관리"
  섹션과 **행위 동등한** 장치 관리 UI를 렌더한다.
- admin에게만 관리 콘텐츠를 렌더한다(접합부 계약 상속).
- 페이지에서 허브(`/admin`)로 복귀할 수 있다(접합부 계약 상속).
- 위 "기존 API 계약"만을 기존과 동일한 방식으로 호출한다 — 요청/응답 계약 무변경.

## 핵심 로직 (동작) — 이전 섹션과의 동등성

아래 각 항목은 이전 "장비(센서) 관리" 섹션에 존재하던 **관측 가능한 동작**이며, 재구성 후
이 서브페이지가 동등하게 보존해야 하는 것이다. 생명주기 의미의 규범 정의는
[[sensor-device-lifecycle]]에 있다.

- **목록 표시** — 활성 장치와 삭제 포함 전체를 각각 기존 `GET /api/devices`,
  `GET /api/devices/all`로 조회해 나열한다. 삭제된 장치는 목록에서 삭제됨 상태로 함께 보인다.
  조회가 실패하면 오류 상태를 표시한다(크래시·빈 화면이 아니다).
- **장치 추가** — `siteId`·`deviceId` 필수(둘 중 하나라도 비면 제출 불가), 별칭 선택. 제출 시
  기존 `POST /api/devices`를 호출한다(생성-또는-재활성; 이미 등록된 미삭제 장치는 409로 안내).
- **별칭 인라인 편집** — 기존 `PATCH /api/devices/{id}`로 별칭을 갱신하며, 빈 별칭 저장은
  별칭 제거를 의미한다.
- **삭제(스티키)** — 기존 `DELETE /api/devices/{id}`(204)로 소프트 삭제하며, 삭제는 유지된다:
  삭제한 장치가 재신호해도 UI 목록에서 자동 복원되지 않는다([[sensor-device-lifecycle]] 계약 2).
- **삭제 확인 안내** — 삭제 확인 시 "삭제한 장치는 자동 복원되지 않으며 되살리려면 재활성이
  필요하다"는 스티키 의미를 정확히 안내한다([[sensor-device-lifecycle]] 계약 4).
- **재출현 안내** — WS `device_reappeared` 수신 시 재출현 안내를 표시하고 재활성·닫기 조작을
  제공한다. 재활성은 기존 `POST /api/devices`를 호출한다([[sensor-device-lifecycle]] 계약 1·2).
- **재활성 어포던스** — 삭제된 장치 행에 복원/재활성 조작을 제공하며, 기존 `POST /api/devices`로
  재활성한다.
- **상태 표시** — 각 장치를 삭제됨 / 경보중 / 온라인 / 오프라인으로 표시한다. 온·오프라인
  경계는 `GET /api/settings`의 `health.sensor_alive_threshold_sec`(미설정 시 60초)로 판정하며,
  `lastSeen`이 null인 장치는 오프라인으로 렌더한다([[sensor-device-lifecycle]] 계약 4).
- **주기적 갱신** — 목록·상태가 수동 새로고침 없이 주기적으로 자동 갱신되어 최신 상태를
  반영한다.

## 검증 단언 (TDD)

관측 가능한 OK/NOK 명제. 라우팅·게이트 계열(A·B·C)은 접합부 계약이 규범 소유이며, 본 잎에서는
이 페이지가 그 계약에 올바로 참여함을 확인한다.

- **A (핵심)** — `/admin/devices` 딥링크/새로고침 진입 시 허브가 아니라 이 장비(센서) 관리
  화면이 렌더된다([[admin-routing-navigation-contract]] B·I 스모크 참조; 정본 판정은 접합부 소유).
- **B (핵심)** — 비-admin·미인증이 `/admin/devices`에 진입하면 관리 콘텐츠가 렌더되지 않고
  기존 게이트 동작으로 유도된다([[admin-routing-navigation-contract]] E 스모크 참조; 정본 판정은 접합부 소유).
- **C (일반)** — 이 페이지에서 뒤로가기(브라우저 back 또는 페이지 내 뒤로 어포던스) 시
  허브(`/admin`)로 복귀한다([[admin-routing-navigation-contract]] C 스모크 참조; 정본 판정은 접합부 소유).
- **D (핵심)** — 목록: 활성 목록은 `GET /api/devices`, 삭제 포함 목록은 `GET /api/devices/all`로
  채워지고, 삭제된 장치는 삭제됨 상태로 목록에 함께 나타난다. 조회가 실패하면 오류 상태가
  표시된다(크래시·빈 화면이 아니다).
- **E (핵심)** — 추가: `siteId`·`deviceId`가 모두 채워져야 제출 가능하고(하나라도 비면 제출
  불가), 별칭은 선택이다. 제출 시 `POST /api/devices`가 호출되며, 이미 등록된 미삭제 장치에
  대한 추가는 409가 사용자에게 안내된다.
- **F (핵심)** — 스티키 삭제: 장치를 삭제하면 `DELETE /api/devices/{id}`(204)가 호출되고, 삭제된
  장치가 재신호해도 활성 목록에 자동 복원되어 나타나지 않는다(삭제됨 유지). 삭제 요청이 실패하면
  오류를 표면화하고 성공으로 오인하지 않는다(#103 무음 삼킴 방지).
- **G (핵심)** — 삭제 확인 안내가 "자동 복원되지 않음 · 되살리려면 재활성"이라는 스티키 의미를
  담고 있다(무음 자동 복원을 암시하지 않는다).
- **H (핵심)** — 재출현: WS `device_reappeared`(payload의 해당 `deviceId`) 수신 시 재출현 안내가
  표시되고 재활성·닫기 조작이 제공되며, 재활성 조작은 `POST /api/devices`를 호출한다.
- **I (일반)** — 재활성 어포던스: 삭제된 장치 행의 복원/재활성 조작이 `POST /api/devices`로
  그 장치를 재활성한다.
- **J (일반)** — 별칭 편집: 인라인 별칭 저장이 `PATCH /api/devices/{id}`를 호출하고, 빈 값 저장은
  별칭을 제거한다.
- **K (일반)** — 상태 배지: 장치가 삭제됨/경보중/온라인/오프라인으로 표시되고, 온·오프라인
  경계가 `GET /api/settings`의 `health.sensor_alive_threshold_sec`(미설정 60초)로 판정되며,
  `lastSeen == null`은 오프라인으로 렌더된다.
- **L (일반)** — 자동 갱신: 목록·상태가 수동 새로고침 없이 주기적으로 갱신되어 변화가 반영된다.
- **M (핵심)** — API 무변경: 이 페이지가 호출하는 엔드포인트 집합은 정확히
  `GET /api/devices`, `GET /api/devices/all`, `GET /api/settings`, `POST /api/devices`,
  `PATCH /api/devices/{id}`, `DELETE /api/devices/{id}`이며, 재구성으로 새 엔드포인트를
  도입하거나 기존 요청/응답 계약을 바꾸지 않는다(각 호출은 동일 admin 인증 헤더를 부착한다).

## 검증 스킵 선언 (선택)

- (없음) — A~L의 UI 렌더·조작 단언은 Playwright 하네스에서 실행되는 **브라우저-게이트 단언**이며
  상시 in-process 테스트에서만 제외될 뿐 스킵이 아니다(접합부 횡단 규칙 참조). M은 정적/네트워크
  관측으로 상시 판정 가능하다.
