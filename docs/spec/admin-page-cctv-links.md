# 임시 CCTV 링크 서브페이지 (/admin/cctv-links) 스펙

> 상태: 살아있는 계약 (living spec) · 독자: 스펙 작성자 / 오케스트레이터
> 역할: 관리(/admin) IA 재구성에서 "임시 CCTV 링크"를 담는 기능별 서브페이지 하나.
> 상위 설계도: [[admin-management-ia-map]] (단위 "임시 CCTV 링크 → /admin/cctv-links").
> 접합부(URL 체계·딥링크·뒤로가기=허브 복귀·admin 게이트)는 재정의하지 않고
> [[admin-routing-navigation-contract]]에만 의존한다.

## 목적 / 의도

관리 영역이 하나의 세로 스크롤 화면에 여러 관리 기능을 담고 있던 것을 "허브 + 기능별
서브페이지"로 재구성하면서, 이전 관리 화면의 **임시 CCTV 링크 섹션**을 독립된 딥링크 가능한
서브페이지(`/admin/cctv-links`)로 이관한다. 이 단위는 **순수 정보구조(IA)·표현 계층 변경**이며,
임시 CCTV 링크의 관측 가능한 동작(링크 생성·목록·만료 표시·폐기)을 **동등하게 보존**한다.
위치와 진입 경로만 바뀌고 기능 계약은 바뀌지 않는다.

상위 맵의 3가지 불변식을 **참조로 상속**한다(여기서 재정의하지 않는다):
① 행위 보존 ② 백엔드·API 무변경 ③ admin 전용 접근 경계 보존.

이전 이 섹션은 임의 인증 사용자에게 보였으나, 불변식 ③에 따라 admin 전용으로 고정된다. 행위
동등성 기준선은 **이전 admin 뷰**이며, 비-admin 접근 상실은 의도된 강화다.

링크의 뷰어 측(`/view/<token>`으로 임시 CCTV를 시청하는 경로)은 이 관리 화면과 무관한 **별도의
기존 라우트**이며, 이 서브페이지의 범위가 아니다. 이 페이지는 링크의 **발급·관리**만 담당한다.

## 언어 · 런타임

브라우저 SPA(React). URL(경로)이 활성 화면의 단일 진실 소스라는 기존 라우팅 원칙을 유지한다
(접합부 계약 소유). 이 페이지는 `/admin/cctv-links` canonical 경로에 마운트되는 서브페이지
컴포넌트 하나다(deliverable 1:1).

## 의존 도구 · 시스템

- **Links API** — `/api/links` (활성 링크 목록·폐기), `/api/links/temp` (임시 링크 생성)
  (기존 계약, 무변경). admin 전용. 요청/응답 계약은 [[interface-web-api]] 계약 9가 소유한다.
- **접합부 계약** — 진입·복귀·admin 게이트·딥링크·미정의 경로 폴백은
  [[admin-routing-navigation-contract]]가 소유한다. 이 페이지는 그 인터페이스에만 의존한다.

## 입력

- **경로 진입:** 허브에서 "임시 CCTV 링크" 항목 선택(드릴다운) 또는 `/admin/cctv-links` 직접
  진입(딥링크·새로고침) — 진입 메커니즘은 접합부 계약 소유.
- **인증 컨텍스트:** admin 역할(접근 경계는 접합부 계약 소유).
- **활성 링크 목록:** `GET /api/links` 응답(각 링크: 서버 발급 식별자·label·생성 시각·만료 시각).
- **사용자 조작:** 링크 생성 요청, 생성된 URL 복사, 링크 폐기 요청/폐기 확인·취소.

## 출력 (계약)

- `/admin/cctv-links`에 마운트되며 admin에게만 임시 CCTV 링크 화면을 렌더한다(게이트는 접합부
  계약을 통해 상속 — 비-admin·미인증은 관리 콘텐츠를 보지 못한다).
- 이전 관리 화면의 "임시 CCTV 링크" 섹션이 제공하던 **링크 생성·목록·만료 표시·폐기 동작을
  동등하게** 제공한다(아래 핵심 로직·검증 단언 참조).
- 링크 생성·폐기는 이전과 동일하게 `POST /api/links/temp`·`DELETE /api/links/{id}`를 호출하고,
  목록은 `GET /api/links`로 조회하며, 요청/응답 계약을 변경하지 않는다.
- 허브로 돌아갈 수 있다(뒤로가기=허브 복귀 — 접합부 계약 소유).

## 핵심 로직 (동작)

이전 임시 CCTV 링크 섹션이 보장하던 관측 가능한 동작을 그대로 보존한다:

- **생성:** 링크 생성 어포던스로 `POST /api/links/temp`(빈 바디)를 호출해 새 임시 링크를
  발급한다. 생성 중에는 생성 어포던스가 진행 상태(중복 요청 차단)로 표시된다. 발급 성공 시
  서버가 반환한 링크 URL을 **읽기 전용 URL 상자**로 노출한다.
- **복사:** 생성된 URL 상자에는 **클립보드 복사 어포던스**가 있어, 조작 시 그 URL을 클립보드에
  복사하고 **일시적 복사됨 피드백**(짧은 시간 뒤 원래 표시로 복귀)을 준다.
- **목록:** 활성 임시 링크를 목록으로 표시하며, 각 항목은 **생성 시각과 만료 시각을 한국시각(KST)
  으로** 표시한다. 로딩 중·오류·빈 목록(활성 임시 링크 없음)의 세 상태를 각각 구분해 표시한다.
  만료·폐기된 링크는 목록에 나타나지 않는다.
- **폐기:** 폐기 어포던스는 곧바로 폐기하지 않고 **폐기 확인 모달**을 거친다. 확인 시에만 해당
  링크를 `DELETE /api/links/{id}`로 폐기하고(성공 시 204), 취소 시 아무 변경도 일어나지 않는다.

'어떻게' 렌더하는지(컴포넌트 구조·상태 관리 방식)는 이 계약이 규정하지 않는다 — 위 관측 가능한
동작만 보존하면 된다.

## 검증 단언 (TDD)

- **A.** admin이 `/admin/cctv-links`에 진입하면 임시 CCTV 링크 화면(생성 어포던스 + 활성 링크
  목록)이 렌더된다. (접근 경계·딥링크 자체는 접합부 A/B/E 스모크 참조이며 정본 판정은 접합부
  소유 — 여기서는 이 경로에서 렌더되는 콘텐츠가 임시 CCTV 링크 관리임을 관측한다.) — 핵심
- **B.** 링크 생성 어포던스를 조작하면 `POST /api/links/temp`(빈 바디)가 호출되고, 응답으로 받은
  링크 URL이 읽기 전용 URL 상자로 표시되며 그 옆에 복사 어포던스가 존재한다. — 핵심
- **C.** 복사 어포던스를 조작하면 표시된 URL이 클립보드에 복사되고, 일시적 "복사됨" 피드백이 나타난
  뒤 원래 표시로 복귀한다. — 일반
- **D.** 활성 링크가 하나 이상 있을 때 목록이 렌더되고, 각 항목에 생성 시각·만료 시각이 KST로
  표시되며 폐기 어포던스가 존재한다. 목록이 비면 "활성 임시 링크 없음" 상태가, 조회 실패 시 오류
  상태가, 조회 중에는 로딩 상태가 각각 구분되어 표시된다. — 핵심
- **E.** 폐기 어포던스는 즉시 폐기하지 않고 폐기 확인 모달을 띄운다. 모달에서 확인해야
  `DELETE /api/links/{id}`가 호출되고(성공 시 204), 취소하면 어떤 폐기 요청도 발생하지 않는다.
  폐기 요청이 실패하면 오류를 표면화하고 성공으로 오인하지 않는다(#103 무음 삼킴 방지). — 핵심
- **F.** 이 페이지가 호출하는 `/api/links`·`/api/links/temp` 요청/응답 계약(경로·메서드·요청 바디
  집합)은 재구성 이전 임시 CCTV 링크 섹션과 동일하다(백엔드·API 무변경 불변식; 각 호출은 동일
  admin 인증 헤더를 부착한다). — 핵심
