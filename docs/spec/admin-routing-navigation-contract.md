# 관리 라우팅·내비게이션 접합부 계약

> 상태: 살아있는 계약 (living spec) · 독자: 스펙 작성자 / 오케스트레이터
> 역할: 관리 허브와 기능별 서브페이지들이 공통으로 기대는 URL 체계·드릴다운·접근 경계 계약.
> [[admin-management-ia-map]]의 모든 단위(허브 + 10 서브페이지)가 이 계약의 인터페이스에만
> 의존한다.

## 목적 / 의도

관리 영역을 "허브(기능 목록) → 기능별 서브페이지(드릴다운)"로 재구성할 때, 모든 단위가 공유하는
**URL 체계·진입·복귀·접근 경계**를 한 곳에서 계약으로 고정한다. 개별 서브페이지는 자신이 무엇을
보여주는지만 알면 되고, "어떻게 도달하고/돌아가고/차단되는가"는 이 계약이 소유한다.

## 언어 · 런타임

브라우저 SPA(React). **URL(경로)이 활성 화면의 단일 진실 소스(SSOT)** 라는 기존 라우팅 원칙을
유지한다 — 화면 전환은 URL을 통해 일어나고, 직접 URL 진입·새로고침·브라우저 뒤로가기가 모두
일관되게 화면을 결정한다.

## 의존 도구 · 시스템

- 브라우저 히스토리(주소·뒤로가기).
- 기존 admin 역할 검사(`isAdmin`) — 이 계약은 새 인증 메커니즘을 만들지 않고, 기존 `isAdmin`
  역할 검사를 재사용하되 이를 **`/admin` 서브트리 전체에 적용되는 라우트-레벨 게이트로 승격**한다.
  이는 이미 admin 전용이던 섹션엔 행위 동등이고, 이전에 임의 인증 사용자에게 보이던 4개
  섹션(health·contacts·현장정보·cctv-links)엔 **의도된 접근 경계 강화**다. 게이트는 이 접합부가
  단독 소유하며, 모든 리프는 그 결과만 상속한다.

## 입력

- 경로 진입: 허브 항목 탭(앱 내 내비게이션), 주소창 직접 진입(딥링크), 브라우저 뒤로/앞으로,
  새로고침.
- 인증 컨텍스트: 현재 사용자의 역할(admin / 비-admin / 미인증).

## 출력 (계약)

- canonical 경로 집합:
  - `/admin` → 관리 허브
  - `/admin/<slug>` → 해당 서브페이지. `<slug>` ∈ { `devices`, `cameras`, `health`,
    `contacts`, `test-alert`, `notify-test`, `system`, `users`, `storage`, `cctv-links` }
- 각 경로는 admin에게만 대응 화면을 렌더한다.
- 상위 4탭 내비(CCTV·경보이력·관리·설정)와 그 canonical 경로(`/cctv`·`/incidents`·`/admin`·
  `/settings`)는 이 재구성으로 **변경되지 않는다**.

## 핵심 로직 (동작)

- **드릴다운:** 허브에서 한 항목을 선택하면 URL이 대응 `/admin/<slug>`로 바뀌고 그 서브페이지가
  렌더된다.
- **딥링크:** `/admin/<slug>`로 직접 진입(주소창/북마크/새로고침)하면 허브를 거치지 않고 해당
  서브페이지가 곧바로 렌더된다.
- **서브라우트 차원 독립:** `/admin/<slug>`는 (a) 게이트·returnTo 허용목록에는 포함되지만,
  (b) 상위 4탭 enum과 **독립된 slug 해석기**가 서브페이지를 마운트한다. slug를 4탭 맵에 넣어
  허브가 렌더되게 하면 NOK다 — slug 해석은 4탭 판정과 별개 차원이다.
- **리프 마운트 계약:** 각 리프는 **props 없는 default export 컴포넌트**를 자신의 canonical
  path에 1:1로 마운트한다. 라우트 테이블·slug↔컴포넌트 등록은 이 접합부(마스터)가 단독
  소유하며, 리프는 라우트 등록 파일을 편집하지 않는다.
- **복귀:** 서브페이지의 **페이지 내 뒤로 어포던스**는 진입 경로와 무관하게 항상 허브(`/admin`)
  로 복귀한다(규범 메커니즘). 브라우저 back은 드릴다운 진입 시 허브로 복귀하며, 딥링크 첫
  진입 시 라우터가 `/admin`을 히스토리에 시드할 수 있다.
- **접근 경계:** 관리 허브 및 모든 `/admin/<slug>`는 admin 전용이다. 이 접합부가 `/admin`
  서브트리 전체에 라우트-레벨 role==admin 게이트(기존 `isAdmin` 검사 재사용)를 적용하며,
  비-admin·미인증 접근은 관리 콘텐츠를 노출하지 않고 기존 게이트 동작(로그인/권한 화면으로
  유도)을 따른다. 이 게이트는 접합부가 단독 소유하고 리프는 결과만 상속한다.
- **미정의 경로 폴백:** slug 집합은 **정확히 위 10개 허용목록**이다. 그 밖의 `/admin/<unknown>`
  은 기존 not-found 폴백(data-view=not-found·탭바 미렌더)으로 처리하며, 빈 화면·크래시로
  떨어지지 않는다. prefix-match(`/admin/`로 시작하면 무엇이든 서브페이지로 간주)는 NOK다.
- **open-redirect 가드 확장:** 로그인 후 복귀 대상(returnTo) 허용목록 가드가 존재하며, 이
  허용목록은 신규 `/admin/<slug>` 경로를 포함하도록 확장하되 앱 외부·임의 URL은 계속 거부한다.

## 횡단 규칙 (cross-cutting)

- **브라우저-게이트 단언 vs 스킵:** UI 렌더·조작을 관측하는 단언은 Playwright 하네스에서
  실행되는 **브라우저-게이트 단언**이며, 상시 in-process 테스트에서만 제외될 뿐 스킵이 아니다
  (needs-browser ≠ skipped).
- **정본 판정 소유:** 게이트·뒤로가기·새로고침·미정의 폴백·returnTo는 이 접합부가 10 slug
  열거(정량자)로 **단독 판정**한다. 각 리프는 이에 대해 **콘텐츠 정체성 스모크 참조**만 가진다 —
  자기 canonical path 진입 시 렌더되는 것이 자기 기능 UI(고유 마커)이고 허브·타 페이지·빈
  화면·크래시가 아님을 확인하는 1개 단언.
- **"행위 동등"의 정의:** 각 리프의 "행위 동등"은 그 리프가 **열거한 관측 단언 집합의 충족**으로
  정의된다. 열거되지 않은 동작은 이 계약의 판정 밖이다.
- **게이트 단언 픽스처(전제):** 게이트 계열 단언(E 등)의 non-vacuous 판정을 위해 이 접합부는
  **비-admin(role≠admin) 픽스처**와 **미인증 픽스처** 두 가지를 전제로 제공한다.

## 검증 단언 (TDD)

- **A.** `/admin` 직접 진입 시 관리 허브가 렌더되고, 10개 서브페이지로 가는 항목이 모두
  존재한다. — 핵심
- **B.** 10개 `<slug>` 각각에 대해 `/admin/<slug>` 직접 진입(딥링크) 시 **허브가 아닌**(크래시·빈
  화면이 아닌) 서브페이지가 렌더되고 URL=SSOT가 유지된다. 특정 페이지 정체성은 각 리프가
  소유한다. — 핵심
- **C.** 임의 `/admin/<slug>`의 **페이지 내 뒤로 어포던스는 진입 경로와 무관하게 항상 `/admin`
  (허브)로 복귀**한다(규범 메커니즘). 브라우저 back은 드릴다운 진입 시 `/admin`으로 복귀한다
  (딥링크 첫 진입 시 라우터가 `/admin`을 히스토리에 시드할 수 있다). — 핵심
- **D.** 허브에서 한 항목을 선택하면 URL이 대응 `/admin/<slug>`로 바뀌고 그 서브페이지가
  렌더된다. — 핵심
- **E.** **라우트-레벨 role==admin 게이트**(기존 `isAdmin` 검사 재사용)에 의해, 비-admin 역할·
  미인증 픽스처가 `/admin` 또는 임의 `/admin/<slug>`에 진입하면 관리 콘텐츠가 렌더되지 않고
  기존 게이트 동작으로 유도된다. 이전에 임의 인증 사용자에게 보이던 섹션도 이제 차단됨(의도된
  강화)을 포함한다. — 핵심
- **F.** 정의된 **10 slug 허용목록 외** `/admin/<unknown>` 진입 시 기존 404 폴백
  (data-view=not-found·탭바 미렌더)이 동작하고, 빈 화면·크래시가 발생하지 않는다.
  prefix-match로 임의 하위경로를 서브페이지로 간주하면 NOK다. — 일반
- **G.** **전제: returnTo 허용목록 가드가 존재한다.** 로그인 returnTo에 신규 `/admin/<slug>`를
  주면 로그인 후 그 경로로 복귀하고, 앱 외부/임의 URL을 주면 거부(무시·기본 경로로 대체)한다.
  — 핵심
- **H.** 상위 4탭 내비 항목과 canonical 경로(`/cctv`·`/incidents`·`/admin`·`/settings`)는
  재구성 후에도 그대로 존재·동작한다. — 일반
- **I.** 임의 `/admin/<slug>`에서 새로고침 후에도 동일 서브페이지가 복원된다(URL=SSOT). — 일반

## 검증 스킵 선언 (선택)

- (없음) — 위 UI 계열 단언은 브라우저-게이트 단언이며 스킵이 아니다(횡단 규칙 참조).
