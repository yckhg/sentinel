## Codebase Patterns
- Stream adapters follow the pattern: Go binary + FFmpeg in alpine:3.19, listening on :8080 with /healthz
- Container naming: sentinel-{source-type}-adapter
- Config files go in /config/ directory, mounted read-only from ./config/
- All adapters depend on streaming service and connect to sentinel-net
- RTMP push destination: rtmp://streaming:1935/live/{streamKey}
- Existing adapters: cctv-adapter (RTSP, passthrough), youtube-adapter (YouTube/file, re-encode)
- AGENTS.md files are gitignored — use `git add -f` when they need to be committed
- Shared FFmpeg base image: build with `docker build -f docker/ffmpeg-base.Dockerfile -t sentinel-ffmpeg-base .` before `docker compose build`

- **Learnings for future iterations:**
  - Docker Compose `deploy.resources.limits` works in non-Swarm mode with recent Docker versions
  - NanoCpus value = cpus × 1e9 (e.g., 2.0 CPU = 2000000000 NanoCpus)
  - Memory value in bytes (e.g., 512M = 536870912)
  - web-frontend was not given limits as it serves static files via nginx (very low resource usage)
  - mosquitto was not given limits as it's a third-party image with minimal resource usage
