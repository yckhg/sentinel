## Codebase Patterns
- CameraManager.Reload() must hold cm.mu.Lock() for the entire read-old + compute-diff + update operation; process kills and goroutine launches go outside the lock
- Notifier service (services/notifier/main.go) has no external Go dependencies — all sanitization/validation must use stdlib only
- Docker services communicate via `sentinel-net` bridge network — internal IPs are in private ranges (172.x)
- Email sending goes: web-backend → notifier `/api/send-email` → SMTP
- Go modules use `go 1.22` with multi-stage Docker builds (golang:1.22-alpine → alpine:3.19)
- `links.go` already uses `crypto/rand` for UUID generation — no external uuid dep needed for random tokens
- Runtime containers don't have Go toolchain — typecheck by running `docker compose build <service>`
