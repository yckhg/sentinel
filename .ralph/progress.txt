## Codebase Patterns
- Go services use `net/http` stdlib with `http.NewServeMux()` and Go 1.22 enhanced routing (e.g., `GET /path`)
- All Go services listen on `:8080` internally, exposed via Docker networking only
- Frontend: Vite + React + TypeScript, built to static files served by nginx
- web-frontend exposed on port 3080 (host port 80 taken by gateway)
- Each service has `/healthz` endpoint for health checks
- Multi-stage Docker builds: Go uses `golang:1.22-alpine` builder + `alpine:3.19` runtime; Frontend uses `node:20-alpine` builder + `nginx:alpine` runtime
- `.gitignore` note: removed `*.ts` wildcard (was blocking TypeScript files) — HLS segments only exist inside containers at runtime
- REST API: auth endpoints at `/auth/*` (public), protected endpoints at `/api/*` (JWT required)
- Internal service-to-service calls use Docker network isolation, not JWT auth
- JSON payloads use camelCase; DB columns use snake_case
- Pagination pattern: `page` + `limit` query params, response includes `{data: [], pagination: {page, limit, total}}`
- MQTT QoS: alerts=2 (exactly once), heartbeats=0 (at most once), commands=1 (at least once)
- hw-gateway is the sole MQTT client; subscribes `safety/+/alert` and `safety/+/heartbeat`, publishes to specific `safety/{siteId}/cmd/restart`
- Heartbeats processed locally in hw-gateway memory only — not forwarded to other services
- In-memory concurrent state uses `sync.RWMutex` + map; return copies (not pointers) from HTTP handlers to avoid races
- Configurable timeouts via env vars with sensible defaults (e.g., `HEARTBEAT_TIMEOUT_SEC` default 30)
- cctv-adapter uses FFmpeg subprocesses for RTSP→RTMP stream forwarding (`-c copy -f flv`)
- Camera config is a JSON file mounted into container at `/config/cameras.json`
- RTMP push to streaming server at `rtmp://streaming:1935/live/{cameraId}`
- Docker images needing external tools (FFmpeg) use `apk add --no-cache` in runtime stage
- web-backend aggregates data from streaming + cctv-adapter + DB to serve public /api/cameras
- Internal HTTP timeouts: 3s connect, 5s read, 10s total
- hw-gateway forwards alerts to notifier and web-backend in parallel; failures are independent
- notifier POST /api/notify returns 200 immediately (async dispatch); KakaoTalk credentials via KAKAO_API_URL/KEY/SENDER_KEY env vars; NHN SMS via NHN_SMS_APP_KEY/SECRET_KEY/SENDER_NO
- Notification fallback chain per-contact: KakaoTalk → NHN Cloud SMS → web-backend system alarm (POST /api/alarms). Empty env vars = channel not configured (skip gracefully)
- streaming service is a hybrid container: nginx-rtmp (RTMP→HLS) + Go API (healthz/streams). nginx proxies API to Go on :8081
- HLS files at `/tmp/hls/{cameraId}/index.m3u8` via `hls_nested on`; served at `/live/{cameraId}/index.m3u8`
- Notifier constructs temp link URLs itself using FRONTEND_URL env var + `/view/{token}` — web-backend returns the token, notifier builds the public URL
- SQLite via `modernc.org/sqlite` v1.34.5 (pure Go, no CGO) — latest versions require Go 1.25+, pin to v1.34.x for Go 1.22
- SQLite pragmas: WAL mode, busy_timeout=5000, foreign_keys=ON
- Migration mechanism: `_migrations` table tracks applied versions; each migration is a numbered struct with SQL; runs in transaction
- DB path configurable via `DB_PATH` env var (default `/data/sentinel.db`)
- `writeJSON(w, status, data)` helper in auth.go for consistent JSON responses — reuse across all handlers
- `golang.org/x/crypto` v0.31.0 for bcrypt — compatible with Go 1.22
- SQLite UNIQUE constraint errors contain "UNIQUE constraint failed" in error string
- JWT via `github.com/golang-jwt/jwt/v5` v5.2.1 — HS256 signing, 24h expiry, custom Claims struct with userId+role
- Auth middleware pattern: separate `apiMux` for /api/* routes, wrapped with `authMiddleware(apiMux)` and mounted on main mux via `mux.Handle("/api/", ...)`
- AuthUser stored in request context via `context.WithValue` — retrieve with `getAuthUser(r)` in handlers
- For admin-only /auth/* routes (outside /api/* middleware), use `requireAdmin(w, r)` inline helper — returns nil and writes error if not admin
- Admin seeding: check if username exists first, skip if already created (idempotent on restart)
- JWT_SECRET env var for token signing; random secret generated if unset (tokens won't survive restarts)
- Temporary links are in-memory only (not in DB) — `tempLinkStore` with `sync.RWMutex`, UUID keys, blacklist set
- TempLinkClaims (linkId) is separate from auth Claims (userId+role) — both use jwtSecret + HS256
- Endpoints that need both external auth AND internal service access: register on main mux (not apiMux) with optional JWT check
- FRONTEND_URL env var in web-backend for constructing temp link URLs (default http://localhost:3080)
- WebSocket via `github.com/gorilla/websocket` v1.5.3 — hub pattern with `sync.RWMutex` for client management
- WebSocket auth: JWT via `?token=` query param; tries parseJWT() first (user), then parseTempLinkJWT() (temp viewer)
- WebSocket broadcast filtering: nil filter = all clients, function filter for role-based (e.g., admin-only for system_alarm)
- POST /api/incidents registered on main mux (not apiMux) — internal service call from hw-gateway, no JWT auth required
- web-backend forwards equipment restart to hw-gateway via HW_GATEWAY_URL env var (default http://hw-gateway:8080)
