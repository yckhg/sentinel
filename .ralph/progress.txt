## Codebase Patterns
- Go services are multi-stage Docker builds (golang:1.22-alpine builder → alpine:3.19 runtime)
- Source code lives at services/{name}/*.go with go.mod in each service dir
- linkStore uses sync.RWMutex for concurrent access protection
- Typecheck via `docker run --rm -v "$(pwd):/app" -w /app golang:1.22-alpine sh -c "go mod tidy && go vet ./..."` from service dir
- Master branch is `master` not `main`
- /data/ is volume-mounted in web-backend container — use it for persistent state files
- hw-gateway has a single main.go with all logic — MQTT handlers, HTTP handlers, heartbeat checker
- equipmentStore uses sync.RWMutex embedded in anonymous struct
