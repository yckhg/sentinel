## Codebase Patterns
- All services use Go stdlib only (no external deps) with `go.mod` at service root
- Health endpoint pattern: `GET /healthz` → `{"status":"ok","service":"<name>"}`
- FFmpeg streams use `-c copy` (no transcoding) and `-f flv` to RTMP destination
- Docker images: `golang:1.22-alpine` builder + `alpine:3.19` runtime
- Config files mounted read-only from `config/` directory
- All services on `sentinel-net` internal Docker network
- Graceful shutdown: SIGTERM → wait → SIGKILL pattern for FFmpeg processes
- Exponential backoff with max 30s for reconnection loops
- Web-backend cameras API matches DB `cameras.name` with streaming `cameraId` — names must be identical
- YouTube stream cameras need DB entries in `cameras` table to appear in CCTV multi-view
- Camera status from cctv-adapter only tracks RTSP cameras; YouTube cameras show "disconnected" status but still stream
- verify.sh must use `docker run` with build images (golang:1.22-alpine, node:20-alpine) since runtime containers don't have dev tools
