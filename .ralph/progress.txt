## Codebase Patterns
- Alpine containers resolve `localhost` to IPv6 `::1`, but Go/nginx listen on IPv4 `0.0.0.0`. Use `127.0.0.1` in healthcheck URLs.
- Go services are multi-stage Docker builds (golang:1.22-alpine builder → alpine:3.19 runtime)
- Source code lives at services/{name}/*.go with go.mod in each service dir
- Typecheck via `docker run --rm -v "$(pwd):/app" -w /app golang:1.22-alpine sh -c "go mod tidy && go vet ./..."` from service dir
- Frontend typecheck: `docker run --rm -v "$(pwd):/app" -w /app node:20-alpine sh -c "npm install --silent 2>/dev/null && npx tsc --noEmit"` from web-frontend dir
- Master branch is `master` not `main`
- /data/ is volume-mounted in web-backend container — use it for persistent state files
- All services expose GET /healthz endpoints — use for Docker healthchecks
- Alpine images have wget (not curl) — use `wget --spider -q` for healthchecks
- Streaming: nginx on port 8080 proxies `/healthz` to Go API on port 8081. RTMP on 1935.
- Web-frontend: nginx on port 80, proxies /api/ and /auth/ to web-backend:8080, /live/ to streaming:8080.
- Web-frontend's `nginx:alpine` crashes if upstream hosts (web-backend, streaming) are not DNS-resolvable at startup.
- cctv-adapter: watchdogWriter wraps io.Writer with atomic.Int64 for lock-free output timestamp tracking
- web-backend DB calls: all handlers use factory pattern `func handleXxx(db *sql.DB) http.HandlerFunc` with `r *http.Request` available for `r.Context()`
- web-backend: `dbCtx(parent)` helper in auth.go provides 5s timeout context — reuse across all files in the package
- Frontend: `fetchWithTimeout()` in `src/utils/fetchWithTimeout.ts` wraps fetch with AbortController timeout — use for all API calls
- Frontend: `isTimeoutError(err)` + `timeoutMessage()` helpers provide consistent Korean timeout error messages
- Frontend: For useEffect with periodic fetches, pass AbortController.signal to fetchWithTimeout and abort on cleanup — replaces `cancelled` flag pattern
- Frontend: `isTokenExpired()` in `src/utils/isTokenExpired.ts` — client-side JWT exp check (no signature verification)
- cctv-adapter: FFmpeg processes use graceful shutdown (SIGTERM → 5s → SIGKILL) via `syscall.SIGTERM`
- streaming: `streamActiveTimeout` var (default 30s, configurable via STREAM_ACTIVE_TIMEOUT env var) controls stream active detection window

## 2026-03-22 - US-008
- **What was implemented**: FFmpeg graceful shutdown via SIGTERM → 5s wait → SIGKILL. Two kill sites updated: (1) watchdog timeout in `manageCameraStream` goroutine, (2) `Stop()` method for container shutdown. Watchdog uses `processDone` channel to detect if process exited within 5s. `Stop()` sends SIGTERM to all processes, sleeps 5s, then SIGKILL any remaining.
- **Files changed**: services/cctv-adapter/main.go
- **Learnings for future iterations:**
  - `syscall.SIGTERM` requires `"syscall"` import — available on all Go targets (Linux/Alpine)
  - `os.Process.Kill()` after process has already exited returns error — safe to call unconditionally as fallback
  - `Stop()` releases mutex before sleeping 5s to avoid blocking status queries during shutdown
  - Watchdog goroutine has access to `processDone` channel (closed when `cmd.Wait()` returns) — efficient signal for graceful exit detection without polling
