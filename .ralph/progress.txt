## Codebase Patterns
- All services use Go stdlib only (no external deps) with `go.mod` at service root
- Health endpoint pattern: `GET /healthz` → `{"status":"ok","service":"<name>"}`
- FFmpeg streams use `-c copy` (no transcoding) and `-f flv` to RTMP destination
- Docker images: `golang:1.22-alpine` builder + `alpine:3.19` runtime
- Config files mounted read-only from `config/` directory
- All services on `sentinel-net` internal Docker network
- Graceful shutdown: SIGTERM → wait → SIGKILL pattern for FFmpeg processes
- Exponential backoff with max 30s for reconnection loops
- Web-backend cameras API matches DB `cameras.name` with streaming `cameraId` — names must be identical
- YouTube stream cameras need DB entries in `cameras` table to appear in CCTV multi-view
- Camera status from cctv-adapter only tracks RTSP cameras; YouTube cameras show "disconnected" status but still stream
- verify.sh must use `docker run` with build images (golang:1.22-alpine, node:20-alpine) since runtime containers don't have dev tools
- Camera CRUD follows same pattern as contacts CRUD: admin-only, partial updates via non-empty fields, `Enabled` uses *bool for explicit false
- cameras table has: id, name, location, zone, stream_key, source_type (rtsp/youtube), source_url, enabled (INTEGER 0/1)
- cameraResponse includes all DB fields plus HLSUrl and Status enriched from streaming/cctv-adapter services
- Internal service-to-service endpoints use `/internal/` prefix (no auth), e.g., `GET /internal/cameras`
- Password/auth handlers for authenticated users use getAuthUser(r) from authMiddleware context; viewer tokens have UserID=0
- cctv-adapter reload: fetches from web-backend, filters to enabled RTSP cameras, diffs with running set
- triggerCCTVReload() is async (goroutine) — fire-and-forget after CRUD operations
- ManagementPage CRUD pattern: state vars → fetch → handlers → JSX (add form + list with inline edit + delete modal)
- Admin-only sections gated by `showAccounts` (isAdmin()) in ManagementPage
- CSS: `mgmt-select` for dropdowns, `mgmt-checkbox-label` for checkbox+label, `mgmt-badge-*` for inline badges
